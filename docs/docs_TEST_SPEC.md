# **✅ テスト仕様書 (Test Specification)**

## **1\. 単体テスト (Unit Testing)**

**対象**: バックエンドAPIロジック (Vitest使用)

| ID | テスト項目 | 前提条件 | 入力値 | 期待する結果 |
| :---- | :---- | :---- | :---- | :---- |
| UT-01 | 正常系: メモ作成 | DB接続可 | title="Test", content="Body" | DBに1件追加され、IDが返る |
| UT-02 | 異常系: 空タイトル | \- | title="", content="Body" | 400エラー、DB追加なし |
| UT-03 | 正常系: メモ一覧取得 | データあり | \- | 配列形式でデータが返る |
| UT-04 | 異常系: 存在しないID更新 | データなし | id=9999, title="Update" | 404エラー |

## **2\. 結合/E2Eテスト (Integration/E2E Testing)**

**対象**: フロントエンド〜DBの通しテスト (Playwright使用)

| ID | シナリオ名 | 手順 | 期待する挙動 |
| :---- | :---- | :---- | :---- |
| E2E-01 | メモのライフサイクル | 1\. ブラウザでアプリを開く 2\. タイトル「買い物」内容「卵」と入力し「追加」押下 3\. 一覧に表示されたメモの「編集」を押下 4\. 内容を「卵、牛乳」に変更し「保存」 5\. 「削除」ボタンを押下 | 1\. 正常に表示される 2\. 一覧に追加される 3\. 入力欄に変わる 4\. 内容が更新される 5\. 一覧から消える |
| E2E-02 | 入力バリデーション | 1\. タイトルを空のまま「追加」ボタンを押下 | 1\. ボタンが非活性(グレー)のままで押せない、またはエラー表示が出る |
